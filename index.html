HTML Form Basics

Text field

Text field syntax example:

<form> <label for="name">First name:</label> <input id="name" type="text" name="name" /> </form>

The example above has two HTML elements: a label and an input.

label: guides users on what to enter and helps with accessibility for people who use screen readers. To connect the label with the input, we use the for attribute in the label, which links to the id attribute in the text field.

input: text fields have special attributes that make them work correctly.

  • id: gives each text field a unique name so the label knows which text field it's associated with.
  • type: defines the kind of input it is. For text fields, the type is set to "text," which means users can enter plain text.
  • name: helps identify the input field when the entire form is sent to a server.

Example of the three additional text field attributes:

<form> <label for="location">Zipcode:</label> <input id="location" type="text" name="zipcode" autofocus="on" maxlength="5" required /> </form>

  1. autofocus: on: sets the first input to autofocus to help draw the user's attention to it. It's best to use this only once on a page.
  2. required: tells the user that this field must be filled in before the form is submitted.
  3. maxlength: number: limits the number of characters added to the text field.

Text area fields

Text area fields allow the user with a multi-line input field so they can enter an unlimited (or limited) amount of characters. These are best used for collecting a longer message.

Text area field syntax example:

<form> <label for="form-message">Your message:</label> <textarea id="form-message" name="message"></textarea> </form>

Attributes to alter the text area appearance and functionality:

<form> <div> <label for="form-message">Your message:</label> <textarea id="form-message" name="message" autofocus="on" required maxlength="200"></textarea> </div> </form>

The example above states that it's a multi-line text field that will store the data within the same message. The autofocus is set to on, the field is required, and only 200 characters can be entered.

  • cols: sets the visible width of a text area in character spaces. Can also use the width attribute in CSS.
  • rows: sets the visible number of lines in a text area in character spaces. Can also use the height attribute in CSS.
  • autofocus: on: sets the first input to autofocus to help draw the user's attention to it. It's best to use this only once on a page.
  • required: tells the user that this field must be filled in before the form is submitted.
  • maxlength: number: limits the number of characters added to the text field.

Fieldset and legend

The fieldset element is used to group together related inputs and labels. It helps with web crawlers and screen readers without visual representation. The legend element is like a title for the fieldset.

Fieldset and legend syntax example:

<form> <fieldset> <legend>Shipping Information</legend> <div> <label for="name">Full Name:</label> <input id="name" type="text" name="name" /> </div> <div> <label for="location">Zipcode:</label> <input id="location" type="text" name="zipcode" autofocus="on" maxlength="5" required /> </div> </fieldset> </form>

The fieldset element is used to group the name and location together.

Placeholder text

The text field and text area field can both use the placeholder attribute to briefly describe the information that is expected to be inputted in the field.

Placeholder text syntax example (with fieldset and legend, text area fields, and text fields):

<form> <fieldset> <legend>Shipping Information</legend> <div> <label for="name">Full Name:</label> <input id="name" type="text" name="name" placeholder="Enter your full name" /> </div> </fieldset> <div> <label for="form-message">Your Message:</label> <textarea id="form-message" name="message" autofocus="on" required maxlength="200" placeholder="How can I help you?"></textarea> </div> </form>

Password field

Password field syntax example:

<form> <div> <label for="password">Password:</label> <input id="password" type="password" name="password" minlength="8" /> </div> </form>

The minlength attribute can be used to force the entry to be at least eight characters in length.

Radio button

Radio buttons allow the user to select only one of the radio button choices available.

Radio button syntax example:

<form> <div> <h2>Favorite Color:</h2> <div> <input id="red" name="color" type="radio" value="red" /> <label for="red">Red</label> </div> <div> <input id="blue" name="color" type="radio" value="blue" /> <label for="blue">Blue</label> </div> <div> <input id="white" name="color" type="radio" value="white" /> <label for="white">White</label> </div> </div> </form>

Favorite Color:

The value attribute is important because that is the data for the selected item that will be submitted with the form data.

The parameters for radio buttons (input, name, and value) can be placed in any order.

When you click the text of the radio button, that specified text is selected because of how the for attribute matches up with the ID of the input element.

Checkboxes

Checkboxes allow the user to select one or more options from the available choices.

Checkbox syntax example:

<form> <div> <h2>Choose your monster's features:</h2> <div> <input name="scales" type="checkbox" id="scales" /> <label for="scales">Scales</label> </div> <div> <input name="horns" type="checkbox" id="horns" /> <label for="horns">Horns</label> </div> <div> <input name="wings" type="checkbox" id="wings" /> <label for="wings">Wings</label> </div> </div> </form>

Drop-down list

A drop-down list, also called a select menu list, is a toggleable menu that allows the user to choose one value from the predefined list. Typically more user friendly than radio buttons when there's a long list of items.

Drop-down list syntax example:

<form> <div> <label for="fonts">Choose a font:</label> <select name="fonts" id="fonts" size="1"> <option value="arial">Arial</option> <option value="times">Times New Roman</option> <option value="comicsans">Comic Sans</option> <option value="papyrus">Papyrus</option> </select> </div> </form>

To create a drop-down list, you have to use the select tag with the option tag. The select tag accepts two attributes:

  1. size: sets the display of the drop-down menu to show a specified number of items. For example, if you want to show a list with 3 items out of 10, you'd set the size to 3 (size="3").
  2. multiple: allows the user to choose multiple sections from the drop-down menu.

Note: It isn't recommended to update the style of an HTML drop-down menu using CSS. It can be done with JavaScript without using this input.

Submit and reset

Submit and reset buttons are similar to links, but buttons are significant in forms. A button will toggle something in the interface, making buttons perfect for submitting or resetting the data in a form.

Submit and reset button syntax example:

<form> <button type="submit">Submit</button> <button type="Reset">Reset</button> </form>

There are three types of buttons:

  1. submit: sends the data in the form to the backend program that collects the data.
  2. reset: clears all of the data that was changed in the existing form.
  3. botton: buttons with type="button" have no default behavior. They are used to trigger client-side functions or scripts that listen to the element's events.

Patterns

The pattern attribute is used to supply patterns that the user's input must match to be valid. A resource for a list of useful patterns can be found at HTML5 Pattern.

Pattern syntax example (validate a phone number):

<form> <div> <label for="phone">Phone Number:</label> <input id="phone" type="text" name="phone" pattern="\d{3}[\-]\d{3}[\-]\d{4}" placeholder="xxx-xxx-xxxx" /> </div> </form>
